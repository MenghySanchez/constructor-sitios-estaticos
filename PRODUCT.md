# Product

## Register

product

## Users

Administradores, creadores de sitios y equipos pequenos que necesitan montar, editar y publicar landing pages estaticas sin depender de una base de datos ni de un flujo tecnico pesado. Usan el panel para iniciar sesion, seleccionar proyectos, construir paginas con bloques, configurar secciones globales, gestionar formularios y publicar carpetas listas para servidor.

## Product Purpose

Static Builder CMS existe para convertir la produccion de landings estaticas en un flujo visual, local y controlable. El exito es que un usuario pueda crear o mantener una landing, guardar cambios en JSON y exportar archivos estaticos con confianza, sin perderse entre opciones tecnicas.

## Brand Personality

Amigable, ligera y practica. La interfaz debe sentirse cercana para usuarios no tecnicos, pero lo bastante solida para trabajo real de produccion. El tono debe ser claro, directo y util.

## Anti-references

Evitar una apariencia SaaS generica, dashboards impersonales, interfaces oscuras agresivas, plantillas tipo admin sin oficio y patrones visuales repetitivos que hagan que el producto parezca intercambiable.

## Design Principles

- Hacer visible el siguiente paso: cada pantalla debe orientar hacia la accion principal sin explicar de mas.
- Reducir friccion tecnica: mostrar rutas, estados y acciones de publicacion con lenguaje claro y contexto suficiente.
- Mantener oficio sin exceso: el sistema visual debe sentirse moderno y propio, no decorativo.
- Priorizar adaptabilidad: el builder debe seguir siendo usable en pantallas medianas y pequenas, aunque el canvas requiera desplazamiento horizontal controlado.
- Dar confianza operativa: guardar, publicar, editar y volver deben sentirse reversibles, evidentes y seguros.

## Accessibility & Inclusion

Objetivo WCAG AA. Priorizar contraste legible, estados de foco visibles, tamanos tactiles comodos, soporte de movimiento reducido y una jerarquia que funcione para usuarios no expertos.
